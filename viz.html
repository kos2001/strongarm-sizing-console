The device

Four phases, one clock, zero static power

The StrongARM latch precharges its nodes high, integrates the input difference onto them, then positive feedback slams the outputs to opposite rails. The trace above is exactly that: two lines held high, splitting at the clock edge.

The sizing problem

Every spec maps to the device that dominates it

There is no closed-form gain equation for a clocked comparator, so the method is to pin each performance metric to the transistor that controls it — then tune only that device when the metric misses.

The agent system

Discover the spec, size it, then close the loop against SPICE

No spec was fixed up front, so the first agent invents realistic ones. Sizing is a first cut; the truth comes from simulation. The tuning loop is the only part an LLM can't do alone — it drives ngspice through the run_sim backend.
