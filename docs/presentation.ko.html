아키텍처

하나의 SPICE 백엔드, 두 개의 문

의존성 없는 파이썬 stdlib 브리지 — 프레임워크 없음. ngspice가 subprocess로 돌아 GIL을 놓으므로, 모든 최적화기와 코너 스윕이 코어 전반에 병렬화됩니다.

  • 비교기와 동일한 최적화 루프 — DE + GP로 목표 주파수를 최소 전력에 맞추고, NSGA-II 전력↔주파수 파레토.
  • 완전한 특성화 — 발진 파형, 튜닝 곡선, 전원 푸싱, PVT(27코너), 위상잡음 + 지터 + FoM.
  • 동일한 사인오프 — GDSII 레이아웃(바이어스 미러 + N단) + DRC, 레이아웃 기생 재시뮬, 엔드투엔드 풀플로우.

연결점

같은 재생 코어, 다른 임무

비교기의 교차결합 래치와 LC-VCO의 −gm 쌍은 동일한 양의 피드백 구조입니다.

  • 비교기는 이걸로 한 번 판정한다 — 레일로 재생.
  • VCO는 같은 −gm으로 계속 발진한다 — 탱크 손실을 상쇄.

최적화 알고리즘

차분진화, 값싸게

전역 탐색

로그공간 DE

소자 폭을 로그공간에서 탐색, 페널티 제약 하에 전력 최소화 — 그리디 하강보다 우수.

sim 절감

GP 대리모델

log(cost) 위의 가우시안 프로세스 모델이 확실히 나쁜 후보를 사전 차단 → SPICE 호출 ~35–45% 감소.

트레이드오프

NSGA-II 파레토

Deb 제약지배 + 혼잡거리 → 도메인별 전력 ↔ 성능 프론트 전체.

모든 후보 평가는 병렬로 실행되고, 승자는 전체 몬테카를로/튜닝 스윕으로 확인합니다.

심화 · VCO 위상잡음

세 가지 관점 — 그리고 정직한 벽

해석

열잡음 L(Δf)

전이당 √(kT·C)/I → 1/f² 영역 + FoM; 가정 코너의 플리커 1/f³ 항.

실측

SPICE trnoise

물리 소자 노이즈 주입, 4시드로 주기 지터 측정 → ~1 dB 이내 일치.

실측

지터 슬로프

σΔt(τ) 누적 → 0.5=열잡음, 1.0=플리커. 열잡음 지배 확인.

정직한 범위  브루트포스 트랜지언트는 오프셋을 ≈ f₀/Ncycles 까지만 분해합니다 — 플리커 코너는 원리적으로 도달 불가이며 PSS/pnoise 엔진이 필요합니다. 도구가 그렇게 명시합니다.
